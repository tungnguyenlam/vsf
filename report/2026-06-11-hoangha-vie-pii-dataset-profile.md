Date: 2026-06-11

profiles its fields, how they map to what our pipeline consumes, integration,

`scripts/profile_dataset.py` (see [Reproducing](#reproducing-this-report)) and

## 3. Fields the dataset ships vs. fields our pipeline uses

This dataset stores its labels **inline inside the text** (`[value]<label>`)
rather than as a separate span column. The loader strips the markup and
reconstructs character spans, so downstream the contract is identical to the
other dataset: `source_text` + `privacy_mask`.

| Dataset column | Our pipeline uses it? | Meaning |
|---|---|---|
| `output` | ⚙️ (parsed) | Vietnamese text with inline `[value]<label>` annotations — the loader's `text_column` |
| `input` | ❌ | Source/prompt text with original (often English) inline labels |
| `total_tokens` | ❌ | Source token-count metadata |
| `text` | ❌ | JSON string summarizing label values |
| `source_text` | ✅ **input** | `output` with inline markup removed (added by loader) |
| `privacy_mask` | ✅ **ground truth** | Char spans reconstructed from `[value]<label>` (added by loader) |
| `split` | ✅ | Normalized split (added by loader) |
| `input_id` | ✅ | Stable id `hoangha_vie_pii:<split>:<source_index>` (added by loader) |
| `inline_label_count` | diagnostic | Count of inline annotations found in `output` |

### Span format

Source markup `[Nguyễn Văn An]<human_name>` is parsed to:

```json
{"start": 0, "end": 13, "value": "Nguyễn Văn An", "label": "human_name"}
```

Offsets are into the **normalized** `source_text`, not the marked-up `output`.

## 4. How it integrates into the pipeline

`HoangHaViePiiDataset` overrides `load()` to do the inline-markup parsing and
local partitioning, but exposes the same interface as every other dataset:

```python
from src.pipeline.Datasets import get_dataset
from src.pipeline.Evaluator import PIIEvaluator

ds = get_dataset("hoangha_vie_pii")
df = ds.load(split="test")                 # normalized: source_text, privacy_mask, split, input_id
evaluator = PIIEvaluator(ds.label_to_presidio)   # dataset owns its own mapping
```

```bash
PYTHONPATH=. python3 scripts/evaluate_pipeline.py \
  --pipeline regex_recall_vie_pii --dataset hoangha_vie_pii --split test --no-log --per-label
```

A tuned pipeline `regex_recall_vie_pii` exists specifically for this corpus; see
`report/2026-06-11-hoangha-vie-pii-cross-dataset.md`.

## 5. Label taxonomy & entity-type counts

Only **7 native labels**, and **all 7 map** to a Presidio type — so unlike
`pii_masking_95k`, nothing is dropped during mapped-type evaluation.

### 5a. Mapped vs. NOT mapped (summary)

Our pipeline targets **9 Presidio types** (`PERSON, LOCATION, ORGANIZATION,
PHONE_NUMBER, EMAIL_ADDRESS, BANK_ACCOUNT, ID, DATE_TIME, MISC`). Unlike
`pii_masking_95k`, **all 7 native labels here are mapped** (100% of spans) — so
nothing is dropped during mapped-type evaluation.

**✅ MAPPED — 7 labels → 7 types:** `id_number`→`ID`, `date`→`DATE_TIME`,
`human_name`→`PERSON`, `company_name`→`ORGANIZATION`, `address`→`LOCATION`,
`email_address`→`EMAIL_ADDRESS`, `phone_number`→`PHONE_NUMBER`.

**❌ NOT present at all:** `BANK_ACCOUNT` (so any `BANK_ACCOUNT` prediction here
is a false positive — it *does* exist in `pii_masking_95k`, which is why
pipelines are tuned per dataset) and `MISC` (reserved, never a ground-truth
label). There are **no** unmapped native labels.

### 5d. Types **not** present in this dataset

Two of our 9 Presidio types have **no ground truth** here:

- **`BANK_ACCOUNT`** — no native label exists, so any `BANK_ACCOUNT`
  prediction counts as a false positive (it has its own label in
  `pii_masking_95k`, which is why pipelines must be tuned per dataset).
- **`MISC`** — reserved type, never a ground-truth label in either dataset.

## 7. Mismatch vs. the pipeline (interpretation)

This corpus is a deliberately *different* benchmark, but its taxonomy is noisier
and broader than `pii_masking_95k`:

- `address` can label URLs and coordinates, not only postal addresses.
- `id_number` can label education-level text in some rows.
- `phone_number` includes non-Vietnamese formats.
- Many rows contain English/mixed-language content after translation.

Treat metrics here as **cross-dataset robustness**, not a clean Vietnamese
production estimate.

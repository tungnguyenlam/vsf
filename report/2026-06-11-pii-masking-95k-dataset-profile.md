# Dataset Report: `pii_masking_95k`

Date: 2026-06-11

Primary evaluation dataset for the Vietnamese PII pipeline. This report profiles
what the dataset contains, how its fields line up with what our pipeline
consumes, how to wire it into the pipeline, and the entity/label counts that
drive evaluation. All numbers were generated by `scripts/profile_dataset.py`
(see [Reproducing](#reproducing-this-report)); they reflect the **full corpus**
(`--split all`, 95,122 rows) unless a split is named.

## 3. Fields the dataset ships vs. fields our pipeline uses

The loader (`BaseDataset.load`) normalizes every dataset to a common contract.
Only **two** columns are load-bearing for the Presidio/regex/LLM pipeline —
`source_text` (the input) and `privacy_mask` (the ground truth). Everything else
is either metadata or a precomputed token encoding our pipeline ignores.

| Dataset column | Type | Our pipeline uses it? | Meaning |
|---|---|---|---|
| `source_text` | str | ✅ **input** | Raw Vietnamese document fed to the pipeline |
| `privacy_mask` | list[dict] | ✅ **ground truth** | Span list — see format below |
| `masked_text` | str | ❌ | Same text with each span replaced by `[LABEL_n]` |
| `language` / `region` / `script` | str | ❌ | Always `vi` / `VN` / `Latn` |
| `uid` | str | ❌ | Stable per-row UUID from the source |
| `tokens` | ndarray[str] | ❌ | Pre-tokenized subwords (SentencePiece-style) |
| `token_classes` | ndarray[str] | ❌ | BIO tags aligned to `tokens` |
| `input_ids` / `attention_mask` | ndarray[int] | ❌ | Encoder inputs for the baked-in tokenizer |
| `offset_mapping` | ndarray[[int,int]] | ❌ | Char span per token |
| `split` | str | ✅ (added by loader) | Normalized split label |
| `input_id` | str | ✅ (added by loader) | Stable id `pii_masking_95k:<split>:<i>` |

> **"Pre-encoded" caveat.** The `tokens`/`token_classes`/`input_ids`/`offset_mapping`
> columns were produced by one *specific* tokenizer baked into the dataset. They
> are useful for training a transformer NER head **only with that same
> tokenizer**. Our pipeline works purely off `source_text` + `privacy_mask`.

### Span format (`privacy_mask`)

```json
{"start": 121, "end": 131, "value": "7204684910", "label": "SO_TAI_KHOAN", "label_index": 1}
```

- `start`/`end` — Python-style char slice (`source_text[start:end] == value`).
- `label` — fine-grained Vietnamese label (110 distinct, see §5).
- `label_index` — 1-based occurrence counter matching the `_n` in `masked_text`.

The loader coerces this through `normalize_privacy_mask` (handles JSON
strings / numpy arrays).

## 4. How it integrates into the pipeline

Each dataset class owns its own `label_to_presidio` mapping; the evaluator reads
that mapping rather than relying on a global dict.

```python
from src.pipeline.Datasets import get_dataset
from src.pipeline.Evaluator import PIIEvaluator

ds = get_dataset("pii_masking_95k")
df = ds.load(split="validation")          # normalized: source_text, privacy_mask, split, input_id
print(ds.presidio_types)                   # the 8 evaluable types
print(ds.unmapped_labels(df))              # labels dropped in this slice

evaluator = PIIEvaluator(ds.label_to_presidio)   # dataset owns the mapping
```

CLI evaluation goes through `scripts/evaluate_pipeline.py` →
`src/pipeline/Pipelines/evaluation.py`, which resolves the registry name, loads
the split, and passes the dataset's mapping to `PIIEvaluator`:

```bash
PYTHONPATH=. python3 scripts/evaluate_pipeline.py \
  --pipeline regex_recall --dataset pii_masking_95k --split validation --no-log
```

## 5. Label taxonomy & entity-type counts

The dataset has **110 distinct fine-grained labels**, but the pipeline targets
**9 Presidio types** (`PERSON, LOCATION, ORGANIZATION, PHONE_NUMBER,
EMAIL_ADDRESS, BANK_ACCOUNT, ID, DATE_TIME, MISC`). The mapping
(`VI_PII_LABEL_TO_PRESIDIO`) covers **23 labels → 8 types**; the remaining 87
labels are out of scope and dropped during mapped-type evaluation.

### 5a. Mapped vs. NOT mapped (summary)

Our pipeline targets **9 Presidio types** (`PERSON, LOCATION, ORGANIZATION,
PHONE_NUMBER, EMAIL_ADDRESS, BANK_ACCOUNT, ID, DATE_TIME, MISC`). Of this
dataset's 110 native labels, **23 are mapped** (53.8% of all spans) and **87 are
not** (46.2% of spans, dropped at eval).

**✅ MAPPED — 23 labels → 8 types:**

| Presidio type | Native labels |
|---|---|
| `PERSON` | `HO_VA_TEN`, `HO`, `TEN`, `TEN_DEM`* |
| `LOCATION` | `THANH_PHO_TINH`, `QUAN_HUYEN`, `PHUONG_XA`, `DUONG_PHO`, `SO_NHA_TOA_NHA`, `QUOC_GIA` |
| `DATE_TIME` | `NGAY`, `NGAY_SINH`, `THANG`, `NAM` |
| `ID` | `MA_NHAN_VIEN`, `MA_GIAO_DICH`, `MA_SO_THUE`, `SO_CCCD_CMND`, `SO_HO_CHIEU` |
| `ORGANIZATION` | `TEN_TO_CHUC`, `TEN_NGAN_HANG` |
| `PHONE_NUMBER` | `SO_DIEN_THOAI` |
| `BANK_ACCOUNT` | `SO_TAI_KHOAN` |
| `EMAIL_ADDRESS` | `DIA_CHI_EMAIL` |

\* `TEN_DEM` is mapped but has 0 occurrences. `MISC` is in our type set but is
**never** a ground-truth label here, so any `MISC` prediction is a false positive.

**❌ NOT MAPPED — 87 labels (dropped at eval):** money/finance (`LOAI_TIEN_TE`,
`SO_TIEN`), job/education (`LINH_VUC_NGHE_NGHIEP`, `CHUC_DANH_CONG_VIEC`),
health (`CHAN_DOAN`, `KET_QUA_XET_NGHIEM`, `DON_THUOC`), credentials
(`MAT_KHAU`, `MA_OTP`, `MA_PIN`, `KHOA_API`, crypto wallets, `DIA_CHI_IP*`),
payment cards (`SO_THE_TIN_DUNG`, `MA_BAO_MAT_THE_CVV`), vehicle (`BIEN_SO_XE`),
demographics (`GIOI_TINH`, `TUOI`, `TON_GIAO`), and more. Full list in §5c.
